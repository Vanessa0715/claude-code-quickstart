CORE DIFFERENCE
兩者都支援 @ 引用檔案、都能直接讀寫本地檔案——操作語法幾乎一樣。
關鍵差異在於修改結果怎麼呈現：桌面版把 diff 顯示在對話視窗裡，你要自己閱讀文字判斷改了什麼；
VS Code 則是在編輯器裡開啟原生 diff 介面，逐行高亮、Accept / Reject 按鈕就在修改旁邊。
任務越複雜、改的檔案越多，這個差距越明顯。
桌面版 CODE 模式 — diff 在對話裡
你：@main.py @settings.json 把 API key 移到 settings.json
Claude：已修改兩個檔案——
main.py 第 12 行移除 API_KEY = "..."
settings.json 新增 "api_key": "..."
→ 對話視窗顯示文字 diff，自己確認後存檔
VS CODE CLAUDE CODE — diff 在編輯器裡
你：@main.py @settings.json 把 API key 移到 settings.json
Claude：找到並修改兩個檔案——
→ VS Code 編輯器直接開啟 diff，紅刪綠增逐行顯示，點 Accept / Reject 確認每一行

━━ 什麼時候用哪個 ━━
用 Claude 桌面版，當你想要⋯
  • Chat快速問答、翻譯、摘要——不需要動任何檔案
  • Chat腦力激盪、探索想法、快速草稿
  • Cowork與他人共用工作空間，一起推進同一個專案（Team / Enterprise 方案）
  • Cowork用 Projects 功能把 PDF、規格書上傳成知識庫，跨對話保留
  • Code偶爾修幾個檔案，不想特地開 VS Code
  • Code執行一次性腳本，路徑和任務都很清楚
用 VS Code Claude Code，當你想要⋯
  • 讓 Claude 修改多個檔案，並看到每一行 diff 再決定要不要接受
  • 正式開發工作：從零建立專案、debug、重構
  • 任務複雜、跨越多個檔案——省 token 又省解釋成本
  • 用 .claude/commands/ 管理可版本控制的快捷指令組合
  • 在修改過程中隨時用 VS Code 的功能跳轉、查文件

Chat 和 Cowork 是日常對話首選，不需要任何進階設定。
需要 Claude 真的動手做事時，Code 模式和 VS Code 都可以——但有兩個實用差異值得注意：

① VS Code 比較省 token：CLAUDE.md 自動載入、@ 精確引用檔案，同樣的任務說明量更少。
② 用量上限行為不同：桌面版 Code 模式的對話視窗一旦達到用量上限就無法繼續，
重要記錄來不及存就消失；VS Code 用量刷新後可以接續原本的工作，不會整個作廢。

偶爾的一次性小任務，桌面版 Code 模式就夠；長時間、複雜的工作，選 VS Code 更穩。

③ 想要「專案背景 + 代理能力」一次到位：把參考文件放進專案資料夾、用 @ 引用，VS Code Claude Code 就能做到類似 Cowork 知識庫的效果。對個人使用者來說，不需要同時維護兩個地方。
兩種情況 Cowork Projects 還是比較方便：① 知識庫以 PDF / Word 文件為主（Projects 預先處理並建索引，不佔對話的 context；VS Code 的 @ 引用會把 PDF 整份內容送進 context，大型文件容易截斷或快速耗盡 token）；② 需要與團隊成員共用同一份知識庫（VS Code 的檔案在本機，要共用得靠 git 或其他方式）。
