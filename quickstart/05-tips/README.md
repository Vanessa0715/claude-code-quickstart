## 問出好答案的關鍵

**給的資訊越具體，答案越好用。**

差的問法：
```
幫我寫一個程式
```

好的問法：
```
幫我做一個單一 HTML 檔案的 webapp，功能是活動預算追蹤。
可以輸入費用項目和金額，顯示已花費和剩餘預算，並用進度條視覺化。
版面要手機也能看，資料用 localStorage 儲存，重新整理後不消失。
```

---

**說清楚你是誰、程度如何。**

```
我完全不會寫程式，幫我用最簡單的方式解釋...
```

```
我是行銷人員，不太懂技術，幫我用非技術的語言說明...
```

---

**結果不好就繼續說。**

不用重新開始，直接補充：
```
這個太複雜了，幫我再簡化一點。
```
```
可以加上繁體中文的說明嗎？
```
```
這個部分我看不懂，再解釋一下。
```

## 常用斜線指令

在對話框輸入 `/` 就會看到可用指令，常用的有：

| 指令 | 功能 |
|------|------|
| `/help` | 查看所有指令 |
| `/clear` | 清空目前對話，重新開始 |

---

## 什麼時候開新對話？

Claude 在同一個對話裡會「記住」前面說過的事，但討論的話題太多時容易混亂。

建議：**換一個新主題就開新對話。** 用左上角的 `+` 按鈕。

---

## Claude 不是萬能的

幾件要注意的事：

- **資訊有截止日期**：Claude 的知識有更新截點，問最新的時事或股價要另外查證
- **做出來要測試**：Claude 給的 HTML 通常可用，但還是要在瀏覽器打開確認，手機也要看一下
- **重要決定要確認**：醫療、法律、財務等問題，Claude 只能參考，不能替代專業建議
