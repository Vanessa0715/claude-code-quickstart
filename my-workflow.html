01
🏗
全域基礎建設
所有專案共享
全域 CLAUDE.md — 寫一次，所有專案都讀得到的個人規則：語言偏好、稱呼方式、回應格式
全域快捷指令（Skills）：常用 prompt 變成一個 /指令，跨專案都能用 /daily-report 一鍵產出今日個人日報（HTML 儀表板） ·  /commit 快速提交當前檔案 ·  /sync-commands 指令有更動時自動同步 Notion
💡 概念：就像手機的「全域設定」vs「單一 App 設定」——全域的每個 App 都繼承，不用一個一個設。
02
🔔
自動化通知系統（Hooks）
自動觸發
Hooks 是「當某件事發生時，自動執行這個動作」的機制——不需要 Claude 自己決定，系統層級觸發
Discord Webhook 整合：Claude 工作狀態即時推送到 Discord，不用一直盯著畫面
需要授權
推送「Claude Code 需要你確認權限，請回來看一下！」
閒置中
推送「Claude Code 閒置中，等你下指令」
認證成功
推送「認證成功，可以繼續了」
Hooks 覆蓋 Claude 的完整生命週期：SessionStart PreToolUse PostToolUse Stop SubagentStart 等
💡 概念：就像你在廚房裝了感應器——鍋子開火自動開抽油煙機，不用人去按。
03
🏢
工作場景 — Lita 台灣 HQ
多 Agent 協作
虛擬 AI 小團隊：在 CLAUDE.md 裡定義 5 個部門角色，Claude 根據任務自動判斷要調用哪個
DATA
📊 數據分析部
INTEL
🔍 競品情報部
COPY
✍️ 文案設計師
SCHED
📅 排班規劃師
REPORT
📋 戰情副官
Sub-agent 並行：說「今日戰情日報」，主 Claude 同時派出多個分身各自執行，REPORT 最後統整——比一件一件做快很多
專案指令： /analyze-data 自動找最新飛書 CSV → 分析（產值 / 異常偵測 / 行動建議）→ 寫入 Notion 當日日報
💡 概念：角色是規則，Sub-agent 是執行者——同一套規則可以同時跑多個人去做。

📖
個人成長場景 — English Learning
學習工具化
CLAUDE.md 定義 AI 家教行為規則：繁中解說、英文出題、主動糾錯、IPA 音標格式⋯⋯不用每次重新說明
雙模式切換：說「工具模式」切到開發助手，說「學習模式」切回家教——用關鍵字控制 Claude 的人格
專案指令：/quiz 自動出題 · /grade 批改作答
Firebase 部署：學習 tracker 是真實 Web App，掛在線上可以跨裝置存取
06
💼
求職場景 — Job Hunt 2026
特定目標作戰室
履歷版本管理規則：CLAUDE.md 規定「逐版留存、不直接覆蓋」，讓 Claude 像版控工具一樣管理履歷迭代
i18n 更新流程：先改 TW 版，改完主動詢問「CN / EN 要一起更新嗎？」確認才同步——防止意外改動
核心亮點強制置頂：CLAUDE.md 定義「量化成果優先、避免空洞描述」，任何履歷修改都自動套用
💡 一個專案 = 一個任務情境。開啟資料夾，Claude 就自動進入對應模式，不用每次重新說明背景。

工具開發場景 — My Planner
用 AI 做 App
設計截圖驅動開發：把 Tiimo App 截圖放進專案資料夾，CLAUDE.md 定義像素級還原規則——Claude 直接對齊設計語言
Agent 角色分工：CLAUDE.md 裡定義 UI/UX 設計師（負責視覺）和前端架構師（負責功能），兩個角色各司其職
輸出：PWA——單一 HTML 檔案，可以「加入主畫面」像 App 一樣打開，沒有工程背景也能做到
