# 05 — 讓 Claude 做的網頁活起來：Firebase + GitHub Pages

> 這個章節教你把 Claude 做出來的網頁，變成一個**真正可以用的工具**：資料存在雲端、跨裝置同步、有自己的網址讓別人打開。

---

## 完整流程

```
Claude 幫你做出網頁（含 Firebase API）
    ↓
Firebase 存放資料（跨裝置同步，改了馬上看到）
    ↓
Git 追蹤版本 → GitHub 存放備份
    ↓
GitHub Pages 發布網址（任何人都能用瀏覽器開啟）
```

---

## Part 1：Firebase — 讓你的網頁記住資料

### Firebase 是什麼？

Firebase 是 Google 的雲端服務，你可以把它想成：

> 你的 HTML 網頁本身只是一張畫布，Firebase 是畫布後面的倉庫——不管誰打開這個網頁，看到的都是同一份最新資料。

對這個課程最有用的功能是 **Firestore**，一個可以直接從 HTML 讀寫的雲端資料庫。

### 實際情境

假設你是餐廳老闆，Claude 幫你做了一個「每日流量觀察儀表板」，你想：
- 在店裡的電腦記錄當天的來客數、熱門品項
- 回到家用筆電打開，資料還在
- 請員工也能輸入，你即時就看得到

這就是 Firebase 的用途。

### 請 Claude 把 Firebase 加進你的網頁

**Step 1：先在 Firebase 建立專案**
```
我想在我的 HTML 網頁裡加入 Firebase，讓資料可以存在雲端。
請告訴我怎麼建立 Firebase 專案，以及怎麼拿到 API 設定碼。
我完全沒用過 Firebase。
```

**Step 2：把 Firebase 加進現有的 HTML**
```
我有一個餐廳流量追蹤的 HTML 網頁，現在資料只存在瀏覽器本機。
請幫我加入 Firebase Firestore，讓資料改成存到雲端，
不同裝置打開這個網頁都能看到同一份資料。

我的 Firebase 設定如下：
[貼上你的 Firebase 設定碼]
```

**Step 3：確認可以跨裝置同步**
```
幫我檢查這個網頁的 Firebase 設定有沒有問題，
然後說明我要怎麼測試跨裝置同步是否正常運作
```

## Part 2：Git + GitHub — 版本控制完整教學

### Git 是什麼？

你用過 Google 文件的「版本紀錄」嗎？可以看到某份文件昨天長什麼樣、誰在什麼時候改了什麼？

Git 就是把這個功能帶給**你整個專案資料夾**的工具：

> 每次你覺得「這個版本不錯」，按一個指令存一個快照（叫做 **commit**）。改壞了，隨時可以回到任何一個快照。

### GitHub 是什麼？

Git 的快照存在你的電腦本機。**GitHub** 是把這些快照上傳到雲端的地方：

1. 電腦壞掉、重灌，都不怕——雲端有備份
2. 下一步可以用 **GitHub Pages** 直接發布成有網址的網站
3. 快照之間的差異一目瞭然，改了什麼隨時查

---

### Step 1：第一次設定（只做一次）

打開 VS Code，確認你在專案資料夾裡，然後叫 Claude 幫你初始化：

```
幫我把這個資料夾初始化成 Git 倉庫，
然後做第一個 commit，說明是「餐廳流量儀表板初始版本」。
我完全沒用過 Git，請一步步帶我做。
```

Claude 會執行 `git init`、`git add`、`git commit` 這幾個指令，你只要確認每個步驟就好。

---

### Step 2：養成 commit 的習慣

Commit 就像**遊戲存檔**——你不會打了一整天才存一次，同樣地，改完一個功能就存一下。

**什麼時候 commit 比較好：**
- 改完一個明確的功能（例如：加了「清空」按鈕）
- 東西跑得起來、感覺這版本不錯
- 要休息前

**每次改完，叫 Claude 幫你存：**

```
我剛把「匯出 CSV」功能加進去了，測試也沒問題。
幫我整理這次改了什麼，寫一個清楚的 commit message 然後存起來。
```

Claude 會看你改了什麼，自動寫出有意義的說明（commit message），比你自己想快多了。

**看現在有哪些尚未儲存的改動：**

```
幫我看一下目前有哪些檔案被修改了還沒 commit，
列出改動摘要，問我要不要一起存起來
```

---

### Step 3：Push 到 GitHub

存在本機的 commit，只有你的電腦知道。**Push** 就是把這些快照上傳到 GitHub。

**第一次推上去（需要先有 GitHub 帳號）：**

沒有 GitHub 帳號的話先問 Claude：
```
我還沒有 GitHub 帳號，請告訴我怎麼免費申請一個
```

有帳號之後：
```
幫我在 GitHub 建立一個新的倉庫，
把這個專案推上去，倉庫名稱叫 restaurant-dashboard。
我的 GitHub 帳號是 [你的帳號名稱]
```

**之後每次 push 就更簡單了：**

```
幫我把最新的 commit 推到 GitHub
```

---

### Step 4：分支——安全的實驗沙盒

想改一個功能，但怕改壞現在跑得好好的版本？

這時候用**分支（branch）**：

> 把現在的版本影印一份，在影印本上實驗。成功了再把改動合回去；失敗了，把影印本丟掉，原版完全不受影響。

**開一個新分支來做實驗：**

```
我想試試把儀表板改成深色主題，但不確定效果好不好，
怕改壞了現在的版本。幫我開一個新分支叫 dark-theme，
讓我在上面試試看，不影響主線。
```

**試完覺得好，合回主線：**

```
dark-theme 分支的深色主題我覺得不錯，
幫我把它合併回 main，然後推到 GitHub
```

**試完覺得不好，直接放棄：**

```
dark-theme 分支的實驗我不想要了，幫我刪掉這個分支，
回到 main 繼續
```

---

### Step 5：回溯——改壞了怎麼辦

這是 Git 最讓人安心的功能：**任何一個 commit 都可以回去。**

**先看看有哪些版本可以回溯：**

```
幫我列出這個專案的 commit 紀錄，
顯示每個版本的時間和說明
```

**回到某個特定版本看看（不刪現在的）：**

```
我看到有一個 commit 說「加入匯出功能」，
我想回去看那個版本的樣子，但不要刪掉現在的進度
```

**確定要回到那個版本（捨棄之後的改動）：**

```
我想回到三個 commit 以前的版本，
那時候的版本比較穩定，之後加的功能我不要了。
請帶我操作，並且確認這個動作不可逆再繼續
```

> **注意**：回溯是不可逆的操作，Claude 每次都會先告訴你影響範圍，確認後才執行。如果你不確定，先開一個新分支備份現在的進度再回溯，比較安全。

## Part 3：GitHub Pages — 發布你的網址

### 什麼是 GitHub Pages？

GitHub Pages 可以把你 GitHub 倉庫裡的 HTML，直接變成一個有網址的網站，完全免費。

網址格式：`https://你的帳號.github.io/倉庫名稱`

### 開啟步驟

1. 進到你的 GitHub 倉庫頁面
2. 點右上角 **Settings**
3. 左側選單找到 **Pages**
4. Source 選 **Deploy from a branch**，Branch 選 **main**，資料夾選 `/ (root)`
5. 點 Save，等約 1 分鐘

不確定怎麼做，叫 Claude 帶你：
```
我已經把專案推到 GitHub 了，
請告訴我怎麼開啟 GitHub Pages，讓這個 HTML 有一個可以分享的網址
```

## 全部串起來之後

你最後會得到：
- 一個有固定網址的網頁，傳給任何人都能打開
- 資料存在 Firebase，不同裝置看到的是同一份
- 每次改版都有記錄，可以回溯

改東西的流程也很簡單：
```
修改 HTML（可以叫 Claude 幫你改）
    ↓
git commit（存版本）
    ↓
git push（推到 GitHub）
    ↓
GitHub Pages 自動更新（約 1 分鐘後生效）
```
