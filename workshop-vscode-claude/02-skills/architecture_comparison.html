Claude.ai Skills / Plugins
管理介面
平台 UI（設定 → Skills）
視覺化管理 無需編輯檔案
作用範圍
跨 project 跨對話
帳號層級，所有專案都能使用
外部整合
部分 Plugin 提供 OAuth 流程

Zapier GitHub Google

本質

Claude Code .claude/commands/*.md
管理介面
直接編輯本地 .md 文字檔
完全控制內容 無 UI
作用範圍
預設：單一專案
放在 .claude/commands/（專案內）→ 只有該專案能用

可擴展：跨所有專案
改放 ~/.claude/commands/（家目錄）→ 所有專案都能呼叫
外部整合

auth 需自行設定（一次，存於本機）
Firebase Bash 任何 MCP

▸Claude Code 讀取 .md 檔，整個內容成為當次對話的系統指令
本質
本地 Markdown 提示詞 + MCP 工具呼叫
同樣能整合外部 API，auth 自行管理（一次設定）

功能 Claude.ai Skills Claude Code Commands
執行效果 ✓ prompt 注入，Claude 執行 ✓ 完全相同
跨 project ✓ 帳號層級，預設跨所有專案 △ 預設單一專案；移至 ~/.claude/commands/ 即可跨專案
跨裝置 ✓ 雲端同步，任何裝置都能用 △ 本地檔案；可用 git / dotfiles 同步
外部 API ✓ 部分 Plugin 有 OAuth，平台代管 ✓ 透過 MCP 達到同等效果，auth 自行設定一次
本地工具存取 ✗ 無法讀寫本機檔案、執行 Bash ✓ 可直接操作本地檔案、終端機、系統工具
管理方式 平台 UI，視覺化操作 直接編輯 .md 檔，改完立即生效
版本控制 ✗ 無 git 整合，無歷史紀錄 ✓ 純文字檔，可用 git 追蹤每次修改
適合場景 輕量使用、需要 OAuth 整合、多裝置共享 深度客製化、需存取本地工具、版本追蹤
━━ 怎麼選 ━━
兩者執行效果完全相同：輸入指令後，Claude 都是把對應的 prompt 載入並執行。
最關鍵的實際差異只有兩點：
選 Claude.ai Skills 如果你需要⋯
· 在多台裝置、多個專案共用同一組指令
· OAuth 驗證由平台幫你管（不想自己設定）
· 簡單場景，不需要讀寫本地檔案
選 Claude Code Commands 如果你需要⋯
· 讀寫本地檔案、執行 Bash 指令
· 精細控制指令內容，隨時修改調整
· 用 git 追蹤 SOP 的歷史變更
