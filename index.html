安裝設定
教材一 · 約 10 分鐘

在開始之前，你需要準備三樣東西：VS Code、Claude 帳號、Claude Code extension。

已經用過 claude.ai 網頁版？ 想知道 VS Code 的 Claude Code 和網頁版差在哪裡，先看這份比較圖：Claude.ai vs VS Code Claude Code ↗
同時用桌面版 Claude 的 Code 模式？ 好消息：兩個 client 讀同一個 .claude/ 資料夾，CLAUDE.md、Skills、設定只要寫一次，兩邊都能用。

Step 1：安裝 VS Code

VS Code 是一個免費的程式編輯器，也是我們要用 Claude 的地方。

  1. 前往 code.visualstudio.com
  2. 點「Download」，選你的作業系統（Mac / Windows）
  3. 安裝完成後打開 VS Code

Step 2：切換成中文介面（選做）

  1. 點左側欄的「擴充功能」圖示，或按 Cmd+Shift+X（Mac）/ Ctrl+Shift+X（Windows）
  2. 搜尋 Chinese Traditional
  3. 找到「Chinese (Traditional) Language Pack」，點「Install」
  4. 右下角跳出提示，點「Change Language and Restart」

已經習慣英文介面？跳過這步也完全沒問題。

Step 3：準備 Claude 帳號

  1. 前往 claude.ai
  2. 點右上角「Sign up」，用 Google 帳號或 email 註冊

關於費用：Claude Code 需要付費方案才能使用。

  • Claude Pro（推薦）：每月約 $20 美元（約 NT$650），訂閱後直接用帳號登入，最省事。
  • API Key：Anthropic 另外申請，按用量計費，適合開發者或用量大的人。

免費帳號無法使用 Claude Code Extension，裝好後登入會看到升級提示。

Step 4：安裝 Claude Code Extension

  1. 打開 VS Code，點左側欄的「擴充功能」圖示（四個方塊），或按 Cmd+Shift+X
  2. 搜尋 Claude Code
  3. 找到 Anthropic 出品的那個，點「安裝」
  4. 安裝完成後，左側欄會出現 Claude 的圖示

Step 5：登入

  1. 點左側欄的 Claude 圖示
  2. 點「Sign in with claude.ai」
  3. 瀏覽器會開啟，允許授權即可
  4. 回到 VS Code，看到對話框出現就代表成功了

確認完成

  • VS Code 左側有 Claude 的圖示
  • 點開後有一個可以輸入文字的對話框

第一次對話
教材一 · 約 10 分鐘

Claude 可以回答問題、整理文字、解釋東西。這一章先從最簡單的開始：讓它幫你整理一段亂掉的文字。

認識介面

打開 VS Code 左側的 Claude 面板，你會看到一個對話框。

  • 直接在底部輸入框打字，按 Enter 送出
  • Claude 會在上方顯示回覆
  • 按 + 可以開新對話（清空記憶）

練習一：直接問問題

在對話框裡輸入任何你好奇的問題，例如：

什麼是程式語言？用一句話解釋給完全沒學過的人聽。

送出後看看 Claude 怎麼回答。

練習二：整理文字

這個資料夾裡有一個 sample.txt 檔案，裡面是一份故意弄亂的工作清單。

  1. 打開 sample.txt，全選（Cmd+A / Ctrl+A），複製
  2. 到 Claude 對話框，輸入：
幫我把以下這份清單整理成有條理的格式，依照類別分組：

[貼上你複製的文字]

練習三：用 @ 直接把檔案丟給 Claude

不用手動複製，Claude Code 有更快的方式：

  1. 在 Claude 對話框輸入 @
  2. 跳出選單，找到 sample.txt 並點選
  3. 補上你的問題：
@sample.txt 這個清單裡哪些事情有截止時間？幫我整理出來。

@ 可以引用檔案、資料夾，甚至整個專案，是 Claude Code 裡最常用的技巧之一。

兩種使用模式

到目前為止你用的都是「對話模式」——你問，Claude 回答，文字只在聊天視窗裡。

Claude Code 還有另一種更強大的模式：「代理模式」——Claude 可以直接幫你讀寫電腦上的檔案、執行指令，不用你自己複製貼上。下一章就會用到。

幫我修程式
教材一 · 約 15 分鐘

這一章要讓 Claude 幫你看一段故意寫錯的程式碼，找出問題並修正。

不需要看懂程式碼，這正是重點：你不用懂，Claude 懂就好。

準備

打開這個資料夾裡的 broken.py 檔案，先滾動看一下，大概知道它在做什麼。（它是一個計算行銷活動費用的小程式，裡面藏了幾個錯誤。）

練習一：讓 Claude 解釋這段程式碼

  1. 打開 broken.py，全選（Cmd+A）
  2. 按右鍵 → Ask Claude，或複製後貼到對話框
  3. 輸入：
這段程式碼在做什麼？用簡單的中文說明，不要用專業術語。

練習二：讓 Claude 找 bug

這段程式碼有沒有錯誤？幫我找出來並說明問題在哪裡。

練習三：讓 Claude 直接幫你改檔案

這是 Claude Code 和一般 ChatGPT 最不一樣的地方：它可以直接修改你電腦上的檔案，不需要你自己複製貼上。

  1. 確認 broken.py 已在 VS Code 裡打開
  2. 輸入：
@broken.py 這段程式碼有幾個錯誤，幫我直接修好它。
  1. Claude 會列出問題，並在畫面上顯示想做的修改（綠色/紅色標出差異）
  2. 確認後點「Accept」讓它直接套用到檔案

這就是「代理模式」： Claude 不是給你答案讓你自己貼，而是直接動手做。適合任何「幫我改、幫我建、幫我刪」的任務。

小觀察

注意 Claude 修改前會：先說明找到幾個問題 → 顯示 diff 預覽讓你看清楚 → 等你確認才執行。你隨時可以點「Reject」拒絕某個修改。

幫我做東西
教材一 · 約 15 分鐘

這一章是最有趣的部分：從零開始，讓 Claude 幫你做一個東西。你不需要知道怎麼寫，只要描述清楚你想要什麼。

怎麼給 Claude 任務？

好的 prompt 通常包含三件事：

  1. 你要做什麼（功能描述）
  2. 限制條件（語言、格式、簡單或複雜）
  3. 預期的結果（要輸出什麼）

範例：

幫我做一個單一 HTML 檔案的 webapp，功能是廠商比價記錄。
可以輸入廠商名稱和報價，新增後顯示成表格，並自動標出最低價。
版面要手機也能看清楚，不需要後端。

練習：挑一個任務試試看

打開 prompt-examples.md，裡面有幾個可以直接使用的任務範本。挑一個你覺得有趣的，貼到 Claude 對話框，看它做出什麼。

如果結果不對怎麼辦？

直接繼續說就好，Claude 會根據你的回饋調整：

這個方向對，但我希望輸出的格式改成清單，不要用表格。
這樣太複雜了，幫我簡化成三個步驟就好。
加上錯誤提示，如果使用者輸入空白要顯示警告。

這種來回調整的過程叫做「迭代」，是使用 Claude 最重要的技巧。

實用技巧
教材一 · 約 10 分鐘

恭喜完成前四章！這裡整理一些讓 Claude 更好用的小技巧。

問出好答案的關鍵

給的資訊越具體，答案越好用。

差的問法好的問法
幫我寫一個程式詳細說明功能、格式、限制、你的程度
幫我做一個單一 HTML 檔案的 webapp，功能是活動預算追蹤。
可以輸入費用項目和金額，顯示已花費和剩餘預算，並用進度條視覺化。
版面要手機也能看，資料用 localStorage 儲存，重新整理後不消失。

說清楚你是誰、程度如何。

我完全不會寫程式，幫我用最簡單的方式解釋...
我是行銷人員，不太懂技術，幫我用非技術的語言說明...

結果不好就繼續說，不用重新開始。

這個太複雜了，幫我再簡化一點。
可以加上繁體中文的說明嗎？

常用斜線指令

在對話框輸入 / 就會看到可用指令：

指令功能
/help查看所有指令
/clear清空目前對話，重新開始

什麼時候開新對話？

建議：換一個新主題就開新對話。 用左上角的 + 按鈕。話題太雜時容易讓 Claude 混淆。

Claude 不是萬能的

  • 資訊有截止日期：問最新時事或股價要另外查證
  • 做出來要測試：Claude 給的 HTML 通常可用，但要在瀏覽器確認
  • 重要決定要確認：醫療、法律、財務只能參考，不能替代專業建議

你已經會的事

  • 在 VS Code 裡使用 Claude Code
  • 用 Claude 整理和處理文字
  • 讓 Claude 找 bug 和修程式
  • 從零開始讓 Claude 建一個小工具
  • 用迭代的方式調整 Claude 的輸出

CLAUDE.md：告訴 Claude 你是誰
教材二 · 約 20 分鐘

沒有 CLAUDE.md：每次對話 Claude 都把你當陌生人，什麼都要重新解釋。

有了 CLAUDE.md：Claude 一開口就知道你是行銷主管、習慣用繁體中文、不喜歡技術術語。

CLAUDE.md 是什麼？

它就是一個純文字檔案，名字就叫 CLAUDE.md，放在你的專案資料夾裡。

Claude Code 在你每次打開對話時，會自動讀取這個檔案，把裡面的設定當作對話的前提。

你可以在裡面寫：

  • 你的職位和工作內容
  • 你希望 Claude 用什麼語氣回答
  • 你的團隊常用的術語或慣例
  • Claude 絕對不能做的事

步驟

Step 1：打開模板

點開這個資料夾裡的 my-claude-md/CLAUDE.md，這是一份已經幫你準備好框架的模板。

Step 2：填寫你的資料

把模板裡用 [ ] 標示的部分換成你自己的資訊。不用寫得很完美，先有內容比較重要。

Step 3：存檔

按 Cmd+S（Mac）/ Ctrl+S（Windows）存檔。

Step 4：測試

你好，根據我的 CLAUDE.md，你知道我是做什麼的嗎？請簡單介紹一下你對我的理解。

如果 Claude 能正確描述你的背景，代表它已經讀到你的設定了。

常見問題

Q：CLAUDE.md 放哪裡？
放在你「用 VS Code 打開的那個資料夾」的根目錄。比如打開的是 my-project/，就放在 my-project/CLAUDE.md。

Q：可以有多個 CLAUDE.md 嗎？
可以。不同專案有不同的 CLAUDE.md，Claude 會讀當前打開的那個。你也可以在家目錄放一個全域的。

Q：裡面要用什麼語言寫？
隨便，中文英文都可以。Claude 都看得懂。

Skills：存下你的常用指令
教材二 · 約 15 分鐘

沒有 Skills：每次要請 Claude 整理今日工作清單，都要打一整段說明。

有了 Skills：輸入 /daily-summary，三秒鐘拿到清單。

Skills 是什麼？

Skills 就是存好的 prompt 快捷鍵。

  • 每個 Skill 是一個 .md 檔案，放在 .claude/commands/ 資料夾裡
  • 檔名就是指令名稱（例如 daily-summary.md → 指令 /daily-summary）
  • 在 Claude 對話框輸入 / 就可以看到所有可用的 Skills
你可能注意到： Claude.ai 網頁版也有「Skills」，和這裡的 .claude/commands/ 是不同層的東西——執行效果相同，但管理方式、作用範圍、能不能存取本地檔案都不一樣。想弄清楚差在哪裡，打開 architecture_comparison.html ↗。

Skills 有兩種層級

層級存放位置使用範圍
個人全域~/.claude/commands/你所有的專案都能用
專案專屬你的專案/.claude/commands/只在這個專案裡出現

動手做：執行現有的 Skill

Step 1：看看 Skill 長什麼樣

打開 .claude/commands/daily-summary.md，看一下裡面寫了什麼。（它就是一段中文 prompt，沒有什麼神奇的語法。）

Step 2：執行它

/daily-summary

按 Enter，看看 Claude 怎麼回應。

Step 3：試著回答 Claude 的問題

Claude 可能會問你今天有哪些事，照實回答就好。

動手做：建立你自己的 Skill

Step 1：想一個你很常用的 prompt

  • 「幫我把這段文字翻譯成正式繁體中文」
  • 「幫我寫一封道歉信，語氣要真誠但不要太肉麻」
  • 「幫我整理這份會議記錄的重點和待辦事項」

Step 2：在 .claude/commands/ 裡建立新檔案

  1. 在左側檔案欄，找到 .claude/commands/ 資料夾
  2. 右鍵 → 新增檔案，檔名用英文，例如 translate-formal.md
找不到 .claude 資料夾？ 它是隱藏資料夾（名稱以 . 開頭）。解法：在 VS Code 上方搜尋列輸入 .claude/commands 就能找到。

Step 3：把你的 prompt 貼進去，存檔

Step 4：測試

在 Claude 對話框輸入 /translate-formal（或你取的名字），確認出現了。

小提示：$ARGUMENTS

Skill 裡面可以用 $ARGUMENTS 這個特殊符號，讓你在執行時傳入補充內容。

幫我把以下文字翻譯成正式繁體中文：

$ARGUMENTS

執行時輸入：/translate-formal 你要翻譯的文字，Claude 就會把輸入的文字填到 $ARGUMENTS 的位置。

MCP 整合 🎯 現場示範
教材二 · 約 15 分鐘 + Dispatch demo

沒有 MCP：「Claude，這是我 Notion 的會議記錄，請幫我整理…」（手動複製貼上）

有了 MCP：「幫我找上週四的會議記錄，整理出待辦事項」——Claude 自己去 Notion 找。

MCP 是什麼？

MCP 全名是 Model Context Protocol，概念很簡單：

MCP = Claude 伸手拿東西的管道

平常 Claude 只能看你貼給它的文字。透過 MCP，Claude 可以主動連接其他應用程式，直接存取裡面的資料或執行操作。

你說：「幫我整理 Notion 的會議記錄」
         ↓
Claude 透過 MCP 連進 Notion
         ↓
Claude 找到相關頁面，整理後回傳給你

方式一：Claude.ai 內建整合（比較簡單）

  1. 打開 Claude.ai → 點右上角頭像 → Settings
  2. 點選 Integrations（整合）
  3. 找到 Notion 或 Google Calendar，點「Connect」
  4. 授權登入即可

連上後試試看：

幫我在 Notion 裡搜尋包含「行銷」關鍵字的頁面
我的 Google Calendar 今天有哪些行程？
幫我在 Google Calendar 新增一個明天下午三點、持續一小時的會議，名稱是「週報」

方式二：Claude Code 專案設定（進階）

如果你想在特定專案裡使用 MCP，可以在專案的 .claude/settings.json 裡設定。打開 examples/mcp-config.json 看範例格式。

注意： 這個方式需要申請 API 金鑰，步驟比較多。建議先用方式一確認概念。

現在可以試什麼？

Notion：

  • 在 Notion 裡找「[你的某個頁面關鍵字]」並幫我摘要重點
  • 幫我在 Notion 新增一個頁面，標題是「2026年下半年目標」，內容先留空

Google Calendar：

  • 幫我看看這週有哪些行程
  • 幫我在週五上午十點加一個「覆盤」的提醒，30分鐘

常見問題

Q：MCP 安全嗎？Claude 看得到我所有 Notion 內容嗎？
你授權哪個工作區，Claude 就只能存取那個範圍。建議用專門的「工作」帳號。

Q：連上後不想用了怎麼辦？
到 Claude.ai → Settings → Integrations，點「Disconnect」就斷開了。

Q：支援哪些工具？
Claude.ai 內建支援：Notion、Google Drive、Google Calendar。透過第三方 MCP，還可以接 GitHub、Jira、Slack 等。

🌟 現場示範：Dispatch 遠端控制

學完 MCP 之後，來試試把它和 Dispatch 結合——這是 Claude 桌面版的遠端連線功能，讓你用手機就能驅動桌面版 Claude 執行任務。

情境：你離開電腦了，但想讓 Claude 幫你把資料整理好上傳

  1. 手機開 Claude app，用同一個帳號登入
  2. 進入 Dispatch 模式（遠端連線到桌面版）
  3. 對手機說：「幫我把這個專案的 README 上傳到 Google Drive 的『Claude 練習』資料夾」
  4. 看桌面版 Claude 自動執行，完成後到 Google Drive 確認

MCP 賦予 Claude 工具（Google Drive）+ Dispatch 讓你遠端下指令 → 人不在電腦旁，工作照樣發生。

注意：Dispatch 需要桌面版保持開啟，並且手機和桌面版登入同一帳號。

打包你的工具組合
教材二 · 約 10 分鐘

概念：你的「Claude 工作包」

你已經學了三個東西：CLAUDE.md（設定背景）、Skills（存下常用指令）、MCP（接上外部工具）。

這三樣合在一起，就是你的「Claude 工作包」——一個資料夾，裡面包含你客製化 Claude 所需的所有設定。

my-workspace/
├── CLAUDE.md                    ← 你的設定
└── .claude/
    └── commands/                ← 你的 Skills
        ├── daily-summary.md
        └── translate-formal.md

把這個資料夾分享給同事，他們放進自己的專案就能用了。換電腦時也只需要複製這個資料夾，設定立刻恢復。

為什麼這很有用？

對個人來說

你把自己的 CLAUDE.md + Skills 整理好，可以在換電腦、換專案時一鍵恢復你的整套設定。

對團隊來說

主管或技術人員做好一個工作包，全團隊複製後大家都有相同的工具和規範，不用每個人自己摸索。

怎麼取得別人做好的工具包？

方法一：從 GitHub 複製

用途搜尋關鍵字
專案管理claude code mcp jira / claude code mcp linear
筆記整理claude code mcp notion / claude code mcp obsidian
開發輔助claude code mcp github / claude code mcp supabase
通訊整合claude code mcp slack / claude code mcp gmail
提示： 取得前確認來源可信賴。MCP 設定會有存取你工具的權限。

方法二：安裝 VS Code Extension

在 VS Code 按 Cmd+Shift+X → 搜尋 claude code，可以找到強化 Claude Code 功能的擴充功能。

注意： VS Code Extension 和 MCP / Skills 是不同層級的東西。Extension 是改 VS Code 本身的功能；MCP 和 Skills 是在 Claude Code 裡客製化 Claude 的行為。

現在可以做的事

  1. 確認 CLAUDE.md 裡的內容是最新的
  2. 看看這幾天建的 Skills，把用不到的刪掉，常用的保留
  3. 把整個 .claude/ 資料夾存一份備份，換電腦時直接複製過去

你現在已經知道的事

  • CLAUDE.md — Claude 的說明書，讓它了解你
  • Skills — 你的個人快捷工具庫
  • MCP — 橋接外部工具的管道
  • 工具包 — 把以上三者打包分享的方式

這四個東西組合在一起，就是「客製化 AI 工作流程」的基礎。

⚡ Firebase + GitHub Pages
教材二 · 加碼自習

這個章節教你把 Claude 做出來的網頁，變成一個真正可以用的工具：資料存在雲端、跨裝置同步、有自己的網址讓別人打開。

完整流程

Claude 幫你做出網頁（含 Firebase API）
    ↓
Firebase 存放資料（跨裝置同步，改了馬上看到）
    ↓
Git 追蹤版本 → GitHub 存放備份
    ↓
GitHub Pages 發布網址（任何人都能用瀏覽器開啟）

Part 1：Firebase — 讓你的網頁記住資料

Firebase 是什麼？

你的 HTML 網頁本身只是一張畫布，Firebase 是畫布後面的倉庫——不管誰打開這個網頁，看到的都是同一份最新資料。

對這個課程最有用的功能是 Firestore，一個可以直接從 HTML 讀寫的雲端資料庫。

請 Claude 把 Firebase 加進你的網頁

Step 1：先在 Firebase 建立專案

我想在我的 HTML 網頁裡加入 Firebase，讓資料可以存在雲端。
請告訴我怎麼建立 Firebase 專案，以及怎麼拿到 API 設定碼。
我完全沒用過 Firebase。

Step 2：把 Firebase 加進現有的 HTML

我有一個餐廳流量追蹤的 HTML 網頁，現在資料只存在瀏覽器本機。
請幫我加入 Firebase Firestore，讓資料改成存到雲端，
不同裝置打開這個網頁都能看到同一份資料。

我的 Firebase 設定如下：
[貼上你的 Firebase 設定碼]

Part 2：Git + GitHub — 版本控制完整教學

Git 是什麼？

Git 就是把「版本紀錄」功能帶給你整個專案資料夾的工具。每次你覺得「這個版本不錯」，按一個指令存一個快照（叫做 commit）。改壞了，隨時可以回到任何一個快照。

GitHub 是什麼？

Git 的快照存在你的電腦本機。GitHub 是把這些快照上傳到雲端的地方：電腦壞掉不怕、可以用 GitHub Pages 發布網站、快照之間的差異一目瞭然。

Step 1：第一次設定（只做一次）

幫我把這個資料夾初始化成 Git 倉庫，
然後做第一個 commit，說明是「餐廳流量儀表板初始版本」。
我完全沒用過 Git，請一步步帶我做。

Step 2：養成 commit 的習慣

Commit 就像遊戲存檔——改完一個功能就存一下。

我剛把「匯出 CSV」功能加進去了，測試也沒問題。
幫我整理這次改了什麼，寫一個清楚的 commit message 然後存起來。

Step 3：Push 到 GitHub

幫我在 GitHub 建立一個新的倉庫，
把這個專案推上去，倉庫名稱叫 restaurant-dashboard。
我的 GitHub 帳號是 [你的帳號名稱]

Step 4：分支——安全的實驗沙盒

我想試試把儀表板改成深色主題，但不確定效果好不好，
怕改壞了現在的版本。幫我開一個新分支叫 dark-theme，
讓我在上面試試看，不影響主線。

Step 5：回溯——改壞了怎麼辦

幫我列出這個專案的 commit 紀錄，
顯示每個版本的時間和說明
注意： 回溯是不可逆的操作，Claude 每次都會先告訴你影響範圍，確認後才執行。不確定的話，先開新分支備份現在的進度再回溯。

Part 3：GitHub Pages — 發布你的網址

網址格式：https://你的帳號.github.io/倉庫名稱

  1. 進到你的 GitHub 倉庫頁面
  2. 點右上角 Settings → 左側選單 Pages
  3. Source 選 Deploy from a branch，Branch 選 main，資料夾選 / (root)
  4. 點 Save，等約 1 分鐘
我已經把專案推到 GitHub 了，
請告訴我怎麼開啟 GitHub Pages，讓這個 HTML 有一個可以分享的網址

全部串起來之後

你最後會得到：有固定網址的網頁、資料存在 Firebase 跨裝置同步、每次改版都有記錄。

修改 HTML → git commit → git push → GitHub Pages 自動更新（約 1 分鐘）
